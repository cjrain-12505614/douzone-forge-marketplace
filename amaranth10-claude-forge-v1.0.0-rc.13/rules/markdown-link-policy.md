# Markdown Link Policy Rule

## 원칙

**모든 마크다운 파일에서 경로·파일을 참조할 때 반드시 클릭 가능한 상대 링크로 작성한다.**

douzone-forge는 모듈·프로젝트·참조자료·세션 간 교차 참조가 매우 많다. 텍스트 단독 표기는 클릭 불가하고, 리팩터링 시 끊어진 링크를 검출할 수 없어 맥락 단절을 유발한다.

## 기본 규칙

1. 경로를 텍스트로만 표기하지 않는다 — 항상 `[표시텍스트](상대경로)` 형식
2. 상대 경로는 현재 파일 위치 기준으로 작성한다
3. 코드블록(```` ``` ````) 안의 경로나 폴더 구조 다이어그램은 예외 (링크 불필요)
4. **설계서 파일명(PDF, PPTX 등)도 반드시 클릭 가능한 마크다운 링크로 작성한다** — 텍스트 단독 표기 금지

## URL 인코딩 규칙 (필수)

파일명·경로에 특수문자가 포함된 경우 반드시 URL 인코딩 처리한다.

| 원문자 | 인코딩 | 사례 |
|--------|--------|------|
| `[` | `%5B` | `[CRM]` → `%5BCRM%5D` |
| `]` | `%5D` | |
| 공백 ` ` | `%20` | `기능 개선` → `기능%20개선` |

## 적용 범위

- douzone-forge 워크스페이스 전체 `*.md` 파일
- `projects/`, `context/`, `tasks/`, `history/`, `문서/INDEX.md`, 스킬 SKILL.md 모두 포함
- 코드블록 내부 경로 및 폴더 구조 트리는 예외

## 좋은 예 / 나쁜 예

```markdown
# 나쁜 예 — 클릭 불가 & URL 인코딩 누락
법무관리/tasks/_current.md에서 확인
설계서: [CRM] CRM 데이터 삭제 기능 개선_v1.0.pdf

# 좋은 예 — 클릭 가능 + URL 인코딩
[법무관리/tasks/_current.md](../법무관리/tasks/_current.md)에서 확인
설계서: [`[CRM] CRM 데이터 삭제 기능 개선_v1.0.pdf`](../../문서/01_신규/%5BCRM%5D%20CRM%20데이터%20삭제%20기능%20개선_v1.0.pdf)
```

## 파일별 상대 경로 기준

| 현재 파일 위치 | douzone-forge 루트로 가려면 |
|--------------|---------------------------|
| `CLAUDE.md` (루트) | `./` |
| `projects/*.md` | `../` |
| `modules/법무관리(LTE)/tasks/*.md` | `../../` |
| `modules/법무관리(LTE)/context/GNB/*.md` | `../../../` |

## 관련 자동화

- 향후 `markdown-link-validator` hook이 PostToolUse에서 끊어진 상대 링크를 검출할 예정 (v0.11.0)
