---

`$ARGUMENTS`에 해당하는 업무 관리 파일을 읽어 현재 진행 업무 맥락을 로드한다.

## 로드 순서

**1단계 — 항상 로드**
`{모듈명}/tasks/_current.md` 읽기

**2단계 — 특정 고도화 건이 지정된 경우**
`{모듈명}/tasks/enhancements/` 아래 해당 파일 탐색 후 읽기
파일명에 인수 키워드가 포함된 파일을 찾는다.

## 인수가 없는 경우

현재 컨텍스트에서 모듈명을 추론해 `_current.md`만 로드한다.

## 로드 완료 후 출력

현재 업무 현황을 아래 형식으로 요약한다.

```
📋 {모듈명} 업무 현황 ({날짜} 기준)

진행 중 ({N}건)
- {업무명}: {상태} | 완료 목표: {날짜} | 담당: {이름}

예정 ({N}건)
- {업무명}: {상태} | 착수 예정: {날짜}

블로커
- {내용}
```

파일이 없으면 `templates/tasks/` 템플릿으로 생성할 수 있다고 안내한다.
