# 파일명 규칙

## 기본 포맷

- `YYMMDD_[프로젝트명] 기능또는메뉴명_문서종류_v1.2.ext`

## 예외 포맷

- 양식, 참조용 샘플, 반복 사용 템플릿은 날짜를 생략할 수 있다.
- 예:
  - `[법무법인가온대응] TC관리엑셀업로드_양식.xlsx`

## 기본 해석

- `YYMMDD`
  - 문서 내부 히스토리의 마지막 수정일
- `[프로젝트명]`
  - 문서 묶음의 공통 프로젝트명
  - 예: `[법무법인가온대응]`
- `기능또는메뉴명`
  - 문서의 실제 대상 기능 또는 메뉴
- `문서종류`
  - `화면설계`, `검수시나리오`, `요구사항정리`, `WBS`, `양식`
- `v1.2`
  - 원문 버전 유지

## 제거 대상

- 제품명 접두
  - `[리걸테크에디션]`
  - `[법무관리]`
- 문서 타입 중복 표현
  - `화면설계서` -> `화면설계`
- 작성자명
  - `(손예진)`, `(이수영)`, `(유지수)` 등

## 정규화 예시

- `채번설정 메뉴 화면설계서` -> `채번설정_화면설계`
- `청구관리 메뉴 기능 개선 화면설계서` -> `청구관리기능개선_화면설계`
- `사건등록 팝업 입력·관리 편의 기능 개선 화면설계서` -> `사건등록팝업입력관리편의기능개선_화면설계`
- `TC관리_엑셀업로드_양식` -> `TC관리엑셀업로드_양식`

## 주의

- 같은 문서군 내에서는 띄어쓰기와 구분 방식을 일관되게 유지한다.
- 너무 긴 기능명은 의미가 보존되는 범위에서만 줄인다.
- 프로젝트 공통 접두는 문서군 전체에서 동일하게 유지한다.
- 날짜가 없는 예외 파일은 `양식`, `샘플`, `참조`처럼 반복 사용 자산인지 먼저 확인한다.
