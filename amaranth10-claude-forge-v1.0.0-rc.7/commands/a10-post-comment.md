# /a10-post-comment 커맨드

ONEFFICE 문서 댓글 패널에 본문/멘션을 자동으로 입력하고 등록한다.
멘션 자동완성 팝업은 신뢰된 네이티브 키 이벤트로만 뜨기 때문에,
Chrome MCP 의 `type`/`key` 시퀀스로 `@이름 ` → `Backspace` 2회 패턴을 사용한다.

## 사용 예

```
/a10-post-comment <url> "PRJ-005 분할 완료. 확인 부탁드립니다." --mention 유지수
/a10-post-comment <url> "{{@유지수}} {{@김철수}} 리뷰 요청드립니다."
/a10-post-comment <url> "작업 완료" --no-confirm
```

## 인자

| 인자/옵션 | 필수 | 설명 |
|-----------|------|------|
| `<url>` | ✅ | ONEFFICE 문서 URL (`gwa.douzone.com/ecm/oneffice/...`) |
| `<body>` | ✅ | 댓글 본문. 본문 내 `{{@이름}}` 플레이스홀더로 멘션 위치 지정 |
| `--mention <이름>` | 선택 | 본문 맨 앞에 해당 이름을 멘션 (body 에 플레이스홀더 없을 때 편의용) |
| `--no-confirm` | 선택 | 등록 직전 확인 단계 생략 (기본은 스크린샷 + 사용자 승인) |

## 실행 절차

### 1. Chrome MCP 연결 확인

연결 안 돼 있으면 즉시 중단하고 확장 설치/연결 안내.

### 2. 대상 탭 확보

`navigate(url)` → 탭 제목에서 ` - ONEFFICE` 를 벗겨 문서명 기록.

### 3. 스킬 `a10-oneffice-comment` 실행

- 댓글 패널 열기 (`#TB_COMMENT_PANEL_0`)
- 입력창 포커스 (`#rpText`)
- body 를 `{{@이름}}` 토큰 기준 split → 일반 텍스트는 `type`, 토큰은 멘션 시퀀스
- 멘션 시퀀스: `type("@이름 ")` → `Backspace` 2회 → `.mentionListBox.type02` 첫 항목 클릭

### 4. 등록 직전 확인

`--no-confirm` 이 없으면 스크린샷을 사용자에게 보여주고 승인 요청.

### 5. 등록 버튼 클릭

DOM 셀렉터 후보 순차 시도 → 실패 시 좌표 `(1411, 290)` fallback.

### 6. 게시 검증

3초 대기 후 댓글 목록 마지막 항목에 방금 본문이 포함되는지 확인. 미확인 시 1회
재조회, 그래도 없으면 경고.

### 7. 결과 보고

문서명·멘션 대상·본문·게시 시각을 요약해서 반환.

## 인식 패턴 (자동 제안)

다음 표현이 등장하면 이 커맨드를 자동 제안·실행한다:

- "원피스 댓글 달아줘"
- "ONEFFICE 댓글 작성해줘"
- "@XXX 멘션해서 댓글 남겨줘"
- "원피스 문서에 댓글 달아"
- "댓글에 @XXX 태그"

## 의존 스킬

- `a10-oneffice-comment` — 실제 실행부 (멘션 시퀀스 + 게시 검증)

## 치명적 주의

- **신뢰된 네이티브 키 이벤트만 동작.** `execCommand`/`value=`/`dispatchEvent` 시도 금지.
- **`@이름` 단독 type 은 IME 조합 문제로 실패할 수 있다.** 공백 포함 `@이름 ` →
  `Backspace` 2회 패턴 고수.
- **공개 업무 문서에 자동 댓글 주의.** 기본 확인 단계를 건너뛰려면 `--no-confirm`
  명시 필요. 인사·계약 등 민감 문서는 수동 작성 권장.
- **동명이인 주의.** 자동완성 결과가 여러 명이면 첫 항목 자동 선택 대신 후보 목록을
  사용자에게 제시한다 (잘못된 사람 멘션 방지).
- **등록 버튼 좌표 `(1411, 290)` 는 1920×1080 / 130% 줌 기준.** 해상도·줌이 다르면
  DOM 셀렉터 후보 먼저 시도.
