# Master Vocab Rule

> 학습 #8 — "마스터 X" 어휘 표준 (옛 "우산 X" → "마스터 X")
> 발효: 2026-05-11 (Phase V V-07-01)
> SSoT: `규칙/프로세스/자비스-운영-룰.md`(forge) §2 학습 #8

## 룰 본문

자비스 산출물 본문에 "마스터 PRJ"·"마스터 계획서"·"마스터 지시서" 어휘를 사용한다. 옛 "우산 X" 어휘는 변천사 보존만 허용한다.

### 어휘 매핑

| 옛 어휘 (금지) | 표준 어휘 |
|---|---|
| 우산 PRJ | 마스터 PRJ |
| 우산 계획서 | 마스터 계획서 |
| 우산 지시서 | 마스터 지시서 |
| 우산 마스터 | 마스터 (중복 표현 제거) |

## 적용 시점·범위

- 시점: 신규 자비스 산출물 작성 (Phase S 이후)
- 범위: 모든 .md 본문 (계획서·지시서·검토의견·완료보고서·종착보고서)
- 예외: 변천사·history·changelog·자비스-운영-룰.md SSoT 자체 (옛 어휘 인용 보존)

## Why

어휘 통일 + 새로운 표준 명확화. Phase S 부터 첫 적용. 옛 Phase Q·R·R+ 우산 지시서 자산은 변천사 보존을 위해 rename 하지 않는다.

## 위반 시 처리

PreToolUse Write/Edit Hook (`hooks/umbrella-vocab-block.sh`) 이 "우산 지시서·우산 계획서·우산 PRJ·우산 마스터" 패턴 검출 시 stderr 경고 (block 하지 않음).

```
⚠️  [umbrella-vocab-block] "우산 X" 어휘 감지 (마스터 X 어휘로 통일 권고)
    학습 #8 — "마스터 X" 어휘 표준
    → "우산" → "마스터" 치환 검토. (block 하지 않음, 알림 전용)
```

### 화이트리스트 (변천사 보존 예외)

- 본문에 "옛 우산"·"변천사"·"보존"·"구 어휘"·"이전 어휘" 동반 표현 → 통과
- `*변천사*`·`*변경이력*`·`*history*`·`*changelog*` 파일명 → 통과
- 자비스-운영-룰.md SSoT 자체 → 통과 (학습 #8 본문 인용)
