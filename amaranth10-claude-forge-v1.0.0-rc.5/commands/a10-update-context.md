---

방금 완료한 작업 내용을 해당 LNB 컨텍스트 파일에 반영한다.

## 실행 순서

**1단계 — 기존 파일 읽기**
`$ARGUMENTS`에 해당하는 컨텍스트 파일을 읽는다.
파일이 없으면 신규 생성 모드로 전환한다.

**2단계 — 업데이트 내용 파악**
현재 대화에서 완료된 작업을 분석해 아래 항목 중 변경된 것을 파악한다.
- 새로 구현된 기능
- 변경된 기능 또는 스펙
- 발견된 이슈 또는 해결된 이슈
- 새로운 결정 사항

**3단계 — 파일 업데이트**
기존 파일의 아래 섹션을 업데이트한다.

```
## 현재 구현 상태
- {기능}: {완료 / 개발 중 / 미착수}

## 최근 변경 이력
- {날짜}: {변경 내용 한 줄 요약}

## 알려진 이슈
- [ ] {미해결 이슈}
- [x] {해결된 이슈}
```

새로 생성하는 경우 feature-planning 스킬의 LNB 파일 형식을 따른다.

**4단계 — 완료 보고**
업데이트된 내용을 간략히 요약해 사용자에게 보고한다.
예: "소송관리/소송등록.md 업데이트 완료. 기본 CRUD → 완료로 변경, 첨부파일 용량 이슈 추가."
