# 워크북 신호와 판별 기준

## 먼저 보는 것

- 시트 수
- 숨김 시트 수
- 시트명 패턴
- 병합셀 개수
- 데이터 검증 개수

## 시트명에서 자주 보이는 신호

- `개발검수`, `설계검수`, `검수 시나리오`
  - 검수 시나리오형 가능성 높음
- `WBS`, `업무 배분`, `진행률`
  - WBS형 가능성 높음
- `오류`, `개선사항`, `현황`
  - backlog형 가능성 높음
- `수익현황(이수영)`처럼 사람 이름이 붙은 숨김 시트
  - 작업용/비교용/중간 산출물 가능성 높음

## 구조 신호

### 검수 시나리오형
- 병합셀이 많다
- `PASS/FAIL/검수불가/검수대기/보완` 드롭다운이 있다
- 한 시트가 하나의 메뉴/기능 주제를 가진다

### 요구사항 WBS형
- 상위 요구사항 묶음과 하위 세부행이 섞여 있다
- 날짜, 진행률, 담당자 열이 길게 이어진다
- `(New-YYMMDD)` 같은 추가 요구가 중간에 있다

### 통합 WBS형
- 월별 일정축이 길게 이어진다
- 한 시트 안에 프로젝트 묶음이 반복된다
- 하단 시트 목록이 모듈/개인별로 나뉜다

### backlog형
- 카테고리 제목 행과 실제 항목 행이 번갈아 나온다
- 링크/출처/접수처가 함께 붙는다
- 우선순위와 일정이 한 줄에 같이 있다

## 해석할 때 주의

- 숨김 시트는 "무시" 대상이 아니라 "미완성/비교/작업용" 자산일 수 있다
- 병합셀 때문에 상위 제목과 하위 행이 물리적으로 분리돼 보여도 한 묶음으로 읽어야 한다
- 엑셀에서 보이는 색상 의미는 직접 파싱하기 어렵기 때문에 텍스트로 확인 가능한 사실만 사용한다
