전체 현황을 4 축 통합 영역으로 보고한다.

## 트리거 (6 항)

기존 3 항:
- "프로젝트 현황 보고"
- "현황 종합"
- "지금까지 어디"

신설 3 항 (C-02 보강):
- "현황 보여줘"
- "지금 어디"
- "진행률"

## 절차

### 0. context 영역 영구 참조

- `프로젝트/_dashboard.md` (PRJ 영역)
- `프로젝트/PRJ-2026-014_*/05_산출물/20260512-작업항목-INDEX.md` §4·§5·§6 (작업 항목 영역)
- `규칙/프로세스/자비스-운영-룰.md` §2 (학습 영역)
- `규칙/프로세스/전체-현황-가시화-표준.md` (4 축 SSoT)

### 1. PRJ 포트폴리오 영역

`프로젝트/_dashboard.md`를 읽어 전체 프로젝트 포트폴리오 현황을 파악한다.

### 2. 상세 영역 분기

사용자가 특정 모듈/프로젝트를 지정했으면 해당 파일을 상세 조회한다:
- 모듈 지정 → `[모듈]/tasks/_current.md` + `_개인/sessions/{모듈}/_current.md`
- 프로젝트 지정 → `프로젝트/PRJ-*.md`의 TASK 목록 + 상세 진행현황

### 3. 모듈 영역

각 모듈의 `module-overview.md`에서 context 파일 현황을 확인한다.

### 4. 4 축 통합 보고

아래 형식으로 보고한다:

```
## 프로젝트 현황 보고

**진행 중 프로젝트**: N건 (리스크 M건)
**마지막 세션**: [모듈명] YYYY-MM-DD

### 축 1 — PRJ 영역
| PRJ ID | 프로젝트명 | 모듈 | 진행률 | 상태 | 리스크 |
|--------|----------|------|--------|------|--------|

### 축 2 — 작업 항목 영역
- 28 작업 항목 완수율: X%
- 11 게이트 완수율: X%

### 축 3 — 학습 영역
- 누적: #1~#34 (#12~#19 예약)

### 축 4 — 사이클 영역
- W-N 진행률: X%

### 즉시 해야 할 것 (다음 TODO)
1. (우선순위 순)
```
