# Agent Dispatch

14 업무별 서브에이전트 + 2 메타 에이전트 (총 16) 디스패치 표준 절차.

본 스킬은 [`규칙/프로세스/에이전트-구성-역할-정의.md`](../../../../../Workspace/douzone-forge/규칙/프로세스/에이전트-구성-역할-정의.md) SSoT 의 디스패치 운영본이다. 표준 정의·이론·매트릭스는 SSoT 인용, 본 SKILL 은 디스패치 절차에 집중한다.

## 호환 매트릭스 (변천사)

기존 5-agent 명명 (하준·서연·도윤·지호·수아) 은 본 14 에이전트로 다음과 같이 호환된다. 변천사 보존을 위한 본 매트릭스이며, 신규 디스패치는 14+2 명명으로 진행한다.

| 기존 명명 | 기존 역할 | 1:1 매핑 | 추가 호환 영역 (영역 확장) |
|---|---|---|---|
| A1 하준 | Researcher | **ORACLE** | — |
| A2 서연 | Planner | **CIPHER** | — |
| A3 도윤 | Designer | **VECTOR** | +PRISM (디자인 시스템) |
| A4 지호 | Architect | **CORE** | +NOVA (프론트엔드) +SYNAPSE (MCP 통합) |
| A5 수아 | Validator | **AEGIS** | +TRACE (검수 시나리오) +PROBE (검수 진행) |

본 호환 매트릭스는 SSoT §3.3 의 동일 매트릭스를 인용한다.

## 디스패치 규칙

본 디스패치 4 규칙은 SSoT §8.1 디스패치 규칙 인용 + Subagent 4 핵심 가치 (§8.2) 정합 보강이다.

### 4 규칙

1. **한 에이전트에 한 가지 목표만** — 복합 목표 금지. Subagent §전문화 가치 일관
2. **병렬 가능하면 병렬로** — 독립 작업은 동시 투입. Subagent §병렬화 가치 일관
3. **프롬프트에 반드시 포함**:
   - 최신 의사결정 사항 (`프로젝트/PRJ-NNNN_*/06_의사결정/` 또는 `decision_log.md` 발췌)
   - 작업 배경과 목적
   - 출력 파일 경로와 형식 (Forge 폴더 표준)
   - **"계획 수립이나 확인 요청 없이 바로 실행하세요"** 문구
4. **결과물은 supervisor 가 취합** — 에이전트 산출을 그대로 전달하지 않고 supervisor (NEXUS 또는 사용자) 가 검토 후 보고

### 보강 규칙 (Subagent 4 가치)

- **도구 제한 명시** — 각 디스패치 시 에이전트별 `tools` 필드 일치 확인 (의도치 않은 동작 위험 감소)
- **컨텍스트 격리 보장** — 부모 컨텍스트 누설 회피 (작업지시서 + 산출 위치만 prompt 에 포함)

## 호출 예시

### 시장조사 — ORACLE 단독

```
당신은 프로젝트의 ORACLE (오라클) 에이전트입니다.
영역: 시장조사·벤치마킹
계획 수립이나 확인 요청 없이 바로 실행하세요.

## 배경
PRJ-2026-XXX — A10 신규 모듈 검토.

## 최신 의사결정
2026-05-12 기술위원회 — Claude Enterprise 단일 시트 도입 확정.

## 임무
A10 신규 모듈 후보 3건의 시장 동향 조사.

## 입력
모듈 후보 명세 (작업지시서 §2).

## 산출
프로젝트/PRJ-2026-XXX/01_기획/20260512-시장동향-3후보.md (Forge 표준)

## AC 검증
- 후보 3건 각 5 항목 이상
- 출처 10건 이상
- ISO 42001 정합성 검토 1건 포함
```

### 다중 병렬 — ORACLE + CIPHER + AEGIS

3 에이전트 독립 작업이라 동시 투입.

- ORACLE: 시장조사
- CIPHER: 요구사항 분석
- AEGIS: 두 산출 정합성 검증 (ORACLE·CIPHER 종료 후 순차)

→ ORACLE · CIPHER 병렬 실행 후 AEGIS 순차 실행.

## 교훈 (실전에서 배운 것)

1. **"바로 실행" 명시 의무** — 에이전트에게 "계획 세워서 확인 받아라" 패턴을 쓰면 불필요한 대기 발생 → "바로 실행"을 반드시 명시
2. **최신 결정 인라인 삽입** — 의사결정 사항을 누락하면 잘못된 가정으로 작업함 → 최신 결정 사항을 프롬프트에 인라인 삽입
3. **하네스 본질 (Initializer + Coding) 적용** — 모든 작업 항목 진입 시 ① Initializer 단계 명시 (작업지시서·INDEX·진행 로그 셋업) + ⑤ Coding 단계 점진 진행 (단일 작업 항목 + AC 검증). 자비스 운영 룰 학습 #20 + SSoT §5 인용
