# context-analyzer 출력 형식 참고

## 분석 결과 요약 출력 형식

파일 생성 전에 먼저 아래 형식으로 구조 요약을 출력해 사용자에게 확인받는다.

```
📋 분석 결과 요약

모듈: {모듈명}
소스: {파일명 또는 "스크린샷"}

감지된 구조:
├── {GNB-1}
│   ├── {LNB-1}  ← 생성 예정
│   └── {LNB-2}  ← 생성 예정
└── {GNB-2}
    └── {LNB-1}  ← 생성 예정

총 {N}개 파일 생성 예정.
확인 필요 항목 {N}개 포함.

생성할까요?
```

## lnb-detail.md 자동 생성 시 표기 규칙

| 항목 | 확실한 경우 | 추론인 경우 | 모르는 경우 |
|-----|----------|----------|----------|
| 기능 목적 | 그대로 작성 | _(추론)_ 표시 | "확인 필요" 작성 |
| 라이선스 | 그대로 작성 | "확인 필요" 작성 | "확인 필요" 작성 |
| 구현 상태 | 그대로 작성 | "미착수 (추정)" | "확인 필요" 작성 |
| 연관 테이블 | 그대로 작성 | _(레이블 기반 추론)_ 표시 | 섹션 생략 |

## 보완 요청 출력 형식

파일 생성 후 아래 형식으로 보완 필요 항목을 안내한다.

```
✅ {N}개 파일 생성 완료

⚠️ 아래 항목은 화면에서 직접 확인되지 않아 추론했습니다. 검토 후 수정해 주세요.

법무관리/context/소송관리/소송등록.md
  - 라이선스 적용 범위 (현재: "확인 필요")
  - 연관 테이블명 (현재: lawsuit_master _(추론)_)

법무관리/context/소송관리/소송현황.md
  - 구현 상태 (현재: 전체 "미착수 (추정)")
```
