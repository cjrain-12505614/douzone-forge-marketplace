**자비스·Claude Code 가 답변 송출 직전 4단계 자가 점검 의무** — 모든 Forge 사용 임직원이 일관된 톤·어휘·단어로 소통받을 권리 보장 영역.

본 룰은 [`답변-언어-톤-표준.md`](../../../douzone-forge/규칙/프로세스/답변-언어-톤-표준.md) §5~§7 (rc.7 신설) 본문 cross-ref + [`자비스-운영-룰.md`](../../../douzone-forge/규칙/프로세스/자비스-운영-룰.md) §2 학습 #28 본문 cross-ref.

## 2. 금지 어휘 5건 + 대체 표현 표

| # | 금지 어휘 | 대체 표현 |
|---|---------|---------|
| 1 | 박혔다 | 기록되었습니다 / 추가되었습니다 |
| 2 | 박음 | 수록되었습니다 / 포함되었습니다 |
| 3 | 박힘 | 포함되었습니다 / 명시되었습니다 |
| 4 | 박아두다 | 보존 / 영구 보관 |
| 5 | 본질 | 핵심 / 의도 / 목적 |
| 6 | 영속화 | 영구히 기록 / 영구 자산 |

> "5 어휘" 라벨로 통칭. 실제 6개 패턴 (박음 계열 4개가 동일 어휘군 단일 카운트, 본질·영속화 각 1건씩).

### 2.1 적용 영역 (사용자 본인 어휘 보호 본문 일관)

- ✅ **적용**: 자비스 답변 본문 + Claude Code 답변 본문 + 워크스페이스 산출물(.md)
- ⛔ **적용 외**: 사용자({이름} 수석) 직접 발화 본문 + `_개인/` 영역 + 외부 컨텐츠 인용 + 변천사 본문

## 3. 자가 점검 절차 4단계

답변 송출 직전 다음 4단계 절차 의무:

```
① 금지 어휘 5건 자가 검색
   - 본문 내 박혔다·박음·박힘·박아두다·본질·영속화 검출
   - 검출 시 §2 대체 표현으로 즉시 교체

② 외국어 약어 첫 등장 시 풀이 점검
   - AC·SSoT·R&R·MCP·PoC 등 영문 두문자 첫 등장 시
   - 한글 풀이 + 약어 병기 패턴 확인 (예: "합격 기준(AC)")

③ 단어 중복 사슬 점검
   - 표 헤더 ↔ 표 열 이름 같은 단어 반복 검출
   - 한 문장·표 한 행 내 같은 단어 2회 이상 반복 검출
   - 검출 시 §2 대체 표현 또는 동의어로 교체

④ 비즈니스 정중 어휘 점검
   - 비격식 어휘 (예: "안 됨"·"꼴까닥") 검출
   - 1인칭 비격식 ("나"·"내가") 검출
   - 정중 어휘로 교체 (보고용 객관 표현)
```

### 3.1 자가 점검 시점 분리

- §3 자가 점검 (`답변-언어-톤-표준.md` §3) = **본문 작성 직전** 4 룰 일반 점검
- §7 자가 점검 (`답변-언어-톤-표준.md` §7) = **답변 송출 직전** 4단계 절차 점검
- 두 절차 의도 분리 — 작성 시점 + 송출 직전 시점 이중 점검 의무
