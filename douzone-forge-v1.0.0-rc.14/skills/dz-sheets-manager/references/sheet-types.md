# 시트 유형 판별 규칙

## 1. 검수 시나리오형

### 자주 보이는 신호
- `통합테스트 ID`
- `테스트 케이스`
- `예상결과`
- `테스트 결과`
- `개발 검수 결과`
- `설계 검수 결과`
- `PASS`, `FAIL`, `검수불가`, `검수대기`, `보완`

### 주로 뽑을 것
- 메뉴/기능 단위 검수율
- 미완료 항목
- 결함내용
- 배포 전 필수 조치

### 권장 산출물
- `history/reports/{date}-{module}-검수요약.md`
- 진행 중이면 `tasks/enhancements/*.md`

### 샘플에서 본 추가 특징
- 상단에 전체 검수율 요약 카드가 있고, 하단에 메뉴별 검수 묶음이 다시 나온다
- `완료설계검수`처럼 날짜가 포함된 상태 문자열이 섞일 수 있다
- 활성 탭 외 탭 목록이 문서 하단에 나타날 수 있다

## 2. 요구사항 WBS형

### 자주 보이는 신호
- 고객사명 + `요구사항`
- `설계`, `개발`, `검수`, `배포`
- `진행률`
- `목표일`, `완료예정일`
- 요구사항별 작업 묶음

### 주로 뽑을 것
- 요구사항별 상태
- 잔여 설계/개발/검수 항목
- 일정 지연 사유
- 관련 메뉴/기능

### 권장 산출물
- `history/reports/{date}-{module}-고객사대응-요약.md`
- `tasks/enhancements/{customer-or-topic}.md`

### 샘플에서 본 추가 특징
- 상위 요구사항 묶음과 하위 요구사항 행이 함께 있다
- `(New-YYMMDD)` 형식의 신규 추가 요구사항이 중간에 끼어든다
- 설계/개발/검수/배포가 한 줄에 이어져 있어 날짜 해석 시 열 의미를 분리해야 한다

## 3. 통합 WBS형

### 자주 보이는 신호
- 프로젝트/업무/담당자/진행률이 한 표에 있음
- 월별 일정축
- 여러 모듈 또는 여러 사람의 탭이 공존

### 주로 뽑을 것
- 모듈별 현재 업무
- 담당자/일정 분포
- 지연 구간
- 고객 대응/공통 운영/개발 축 구분

### 권장 산출물
- `history/reports/{date}-{module}-운영WBS-요약.md`
- `tasks/_current.md` 반영

### 샘플에서 본 추가 특징
- 시트 제목과 활성 탭명이 다를 수 있다
- 예: 시트 제목은 전체 WBS인데, 실제 본문은 `CRM WBS`처럼 특정 탭일 수 있다
- 하단 탭 목록에서 모듈별 연도 탭을 같이 확인해야 전체 구조가 보인다

## 4. 오류/개선 backlog형

### 자주 보이는 신호
- `오류`, `개선사항`, `우선순위`
- `출처`, `링크`, `요청일`
- 메뉴별 backlog 목록

### 주로 뽑을 것
- 기능별 개선 묶음
- 반복 이슈
- 고객/내부 출처
- 우선순위가 높은 backlog

### 권장 산출물
- `history/reports/{date}-{module}-개선백로그-요약.md`
- `tasks/enhancements/*.md`

### 샘플에서 본 추가 특징
- 카테고리 제목만 있는 줄과 실제 backlog 데이터 줄이 교차한다
- `프로젝트 상세`, `원피스`, `사업계획서` 같은 링크/출처 행이 바로 뒤따를 수 있다
- 우선순위 외에 `오류`, `개선`, `프로세스 필요` 같은 분류 표현이 섞인다

## 분류가 애매할 때

- 탭 상단 요약 카드가 있고 하단에 테스트 케이스가 이어지면 `검수 시나리오형`
- 일정축과 담당자가 중심이면 `WBS형`
- 메뉴/이슈/우선순위가 중심이면 `backlog형`
- 한 시트 안에 여러 유형이 섞여 있으면 탭별로 따로 분류한다
