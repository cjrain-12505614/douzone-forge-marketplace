# 최종본 선별과 삭제 후보 규칙

## 1. 먼저 묶는 기준

- 같은 기능 또는 메뉴
- 같은 문서종류
- 같은 프로젝트 묶음

## 2. 최종본 후보

아래 순서로 본다.
1. 더 높은 버전
2. 내부 히스토리의 더 최신 수정일
3. 파일명은 비슷하지만 실제 내용이 더 많거나 더 최신인 문서

## 3. 삭제 후보

아래를 모두 만족하면 삭제 후보로 둘 수 있다.
- 같은 문서군에 속한다
- 상위 버전 문서가 있다
- 부록이나 별도 목적이 아닌 구버전이다

## 4. 보류

아래 중 하나면 바로 삭제하지 않는다.
- 제목은 비슷한데 기능 범위가 다르다
- 버전은 낮지만 파일 크기나 내용 구조가 크게 다르다
- 내부 히스토리 표를 아직 확인하지 못했다
- `상담료` / `상담료(상담금액)`처럼 표현 차이가 의미 차이일 수 있다

## 5. 날짜 추출 규칙

- 1순위: 문서 내부 `Document History`, `문서 이력`, `개정 이력`의 마지막 수정일
- 2순위: 내부 본문에서 명확히 확인되는 최종 수정일
- 참고값: `pdfinfo`의 `ModDate`, `CreationDate`

## 6. 답변 형식

- `현재 파일명`
- `제안 파일명`
- `상태`
  - `보관본`
  - `삭제후보`
  - `보류`
- `근거`
