## 2. 요청 흐름 5 단계 진입

### 단계 1 — 사용자 요청 본문 청취

- 사용자 본문 영역 청취
- 4 항목 영역 확인 (요청자·요청 영역·기대 결과·우선순위)

### 단계 2 — 요청 분류

SSoT §2 매트릭스 일관:
- 운영 (Forge 자체 운영) — H
- 개선 (답변 톤·기능 보강) — M
- 문의 (사용 방법·환경 점검) — L
- 이슈 (오류·장애 신고) — C

### 단계 3 — 템플릿 채움

- `_운영/요청/_템플릿.md` 본문 골격 활용
- 4 항목 영역 본문 영역 (사번 미노출 룰 일관 — G-03 §4 인용)

### 단계 4 — 영구 수록

- 영역: `_운영/요청/{YYYYMMDD}-{본문}.md` 영구 자산화
- 표준 명명 영역 (한글·영문 혼합)

### 단계 5 — 14 에이전트 위임 안내

- S-09 다층 자비스 cross-ref (SSoT §5 인용)
- 분류별 위임 영역 안내
