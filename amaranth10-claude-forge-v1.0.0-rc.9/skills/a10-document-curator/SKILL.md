# 문서 큐레이터 (Document Curator)

신규 유입 문서를 바로 자산화하지 않고, 먼저 이름 규칙과 버전 구조를 정리하는 스킬이다.
파일명 정리, 최종본 선별, 삭제 후보 분리, 날짜 기준 확인을 한 흐름으로 다룬다.

## 사용할 때

- `문서/01_신규` 같은 유입 폴더를 정리할 때
- 파일명이 제각각이라 표준명으로 바꿔야 할 때
- 같은 문서의 v1.0, v2.0, v3.0이 섞여 있을 때
- 내부 히스토리의 마지막 수정일을 파일명 날짜로 써야 할 때
- 자산화(context-analyzer) 전에 먼저 정리가 필요할 때

## 기본 원칙

1. 파일을 먼저 지우지 않는다
2. 같은 문서군을 묶고 최종본 후보를 먼저 고른다
3. 날짜는 가능하면 문서 내부 히스토리의 마지막 수정일을 기준으로 한다
4. 내부 히스토리 확인이 안 되면 메타데이터 날짜는 참고만 하고 보류로 둔다
5. 파일명 변경과 삭제는 반드시 근거와 함께 제안한다

## 작업 순서

### 1. 폴더 인벤토리 만들기

먼저 `scripts/inventory_documents.py`로 폴더 내 파일을 훑는다.

```bash
python3 "${CLAUDE_PLUGIN_ROOT}/skills/document-curator/scripts/inventory_documents.py" "{대상폴더경로}"
```

이 단계에서 확인:
- 파일 수, 확장자별 분포
- 버전 표기 유무
- 같은 문서군 후보
- 최종본 후보와 구버전 후보

### 2. 날짜 기준 확인

- PDF는 내부 히스토리 표의 마지막 수정일을 우선 확인한다
- 텍스트 추출이 되면 `pdftotext`로 확인한다
- 이미지형 PDF면 Read 도구로 시각적 확인을 고려한다
- 그래도 확인이 안 되면 `pdfinfo` 날짜는 참고만 남기고 보류로 둔다

날짜 확인과 예외 기준은 `references/triage-rules.md`를 본다.

### 3. 표준 파일명 만들기

파일명 규칙은 `references/naming-rules.md`를 따른다.

기본 구조:
```
YYMMDD_[프로젝트명] 기능또는메뉴명_문서종류_v1.2.ext
```

예:
```
260317_[법무법인가온대응] 채번설정_화면설계_v3.0.pdf
```

### 4. 최종본/삭제후보/보류 분리

| 분류 | 조건 |
|------|------|
| 보관본 | 단건 문서 또는 최종 버전 |
| 삭제후보 | 같은 문서군 + 상위 버전 존재 + 구버전 |
| 보류 | 제목 비슷하지만 내용 차이 의심 / 히스토리 미확인 |

### 5. 최종 정리

최종 답변에는 아래를 포함한다:
- 검토한 폴더 경로
- 문서군 요약
- `현재 파일명 → 제안 파일명` 목록
- 삭제후보 목록 (근거 포함)
- 보류 목록
- 추가 확인이 필요한 문서

## 출력 규칙

- 삭제는 바로 실행하지 말고 먼저 후보로 제시한다
- 날짜 기준을 못 확인한 문서는 그 사실을 명시한다
- 프로젝트 접두, 문서종류 어휘, 기능명 표기는 한 번 정하면 폴더 전체에 일관되게 맞춘다

## context-analyzer와의 관계

document-curator → context-analyzer 순서로 사용한다.
1. document-curator로 문서 정리 (파일명 표준화, 버전 선별)
2. context-analyzer로 정리된 문서를 자산화 (context/ 파일 생성)

## 참고 파일

- 파일명 규칙: `references/naming-rules.md`
- 최종본/삭제후보 판단 규칙: `references/triage-rules.md`
- 폴더 인벤토리 스크립트: `scripts/inventory_documents.py`


## 마크다운 링크 규칙 (필수)

마크다운 파일 작성·업데이트 시 **모든 경로·파일 참조는 클릭 가능한 상대 링크**로 작성한다.

```markdown
# 나쁜 예 (클릭 불가)
법무관리/tasks/_current.md에서 확인

# 좋은 예 (클릭 가능)
[법무관리/tasks/_current.md](../법무관리/tasks/_current.md)에서 확인
```

- 상대 경로는 현재 파일 위치 기준
- 코드블록 안의 폴더 구조 다이어그램은 예외 (링크 불필요)
- 상세 규칙은 douzone-forge CLAUDE.md의 "마크다운 링크 표기 규칙" 섹션 참조
