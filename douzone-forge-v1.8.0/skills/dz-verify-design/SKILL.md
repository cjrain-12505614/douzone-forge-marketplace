# 개발 계획·개발설계 검증·확정 게이트 (dz-verify-design)

> 용어: AEGIS(정합성 검증 전담 서브에이전트 — 읽기·검색만, 변경 불가) · 추적 키(요구사항↔화면↔개발계획↔개발설계↔구현↔검수를 같은 식별자로 잇기) · 품질 게이트(완료 선언 전 전수 통과 체크리스트) · 제안→확정(자동 산정·검증은 제안, 확정은 사람) · 비강제(opt-in, 실행 여부는 선택)
>
> **SSoT(단일 출처)**: 검증 기준 = [`규칙/프로세스/개발-계획-수립-표준.md`](../../../../../Workspace/douzone-forge/규칙/프로세스/개발-계획-수립-표준.md)(D-06 §9) + [`규칙/프로세스/개발설계-산출물-작성-표준.md`](../../../../../Workspace/douzone-forge/규칙/프로세스/개발설계-산출물-작성-표준.md)(D-08 §9). 검증 주체 = AEGIS(`.claude/agents/dz-aegis.md`). 본 스킬은 ⑤ 검증·확정 단계를 **실행**하는 절차다.

## 0. 한 줄 정의

개발 계획(D-06)·개발설계(D-08) 산출물을 **작성자와 분리된 AEGIS**가 독립 검증한다 — 추적 키 grep 조인 + 품질 게이트로 누락·모순·형식 위반을 검출하고 **제안→확정 가능 여부**를 판정한다. 파이프라인 ⑤ 산출물 검증·확정 게이트.

## 1. 발동 / 비강제

- 사용자가 "개발 계획/개발설계 검증"·"정합성 확인"·"확정 가능한지" 요청 시. 또는 dz-devplan-write(③④)·dz-devdesign-write가 작성 직후 호출.
- **비강제(opt-in)**: 실행 여부는 선택. 단 실행하면 D-06 §9·D-08 §9 게이트 전수로 판정(부분 검증 금지).
- 작성↔검증 분리(MUST): 작성한 주체(ATLAS·BLUEPRINT)가 자기 산출을 확정하지 않는다. AEGIS 독립 검증 → 사람 확정.

## 2. 검증 전 체크리스트

1. **대상 산출물 경로** — 개발 계획서(`Amaranth10/{모듈}/history/plans/{모듈}-{기능slug}-개발계획.md`) 또는 개발설계(`Amaranth10/{모듈}/개발설계/{모듈}-{기능slug}-개발설계.md`). 둘 다면 연계 검증(개발 계획 produces ↔ 개발설계 항목 ID).
2. **선행 추적 입력** — 요구사항(D-05 REQ/AC)·화면설계(D-07 SCR) 식별자 확보(조인 기준).
3. **기준 표준** — D-06 §9·D-08 §9 품질 게이트.

## 3. 절차

```
① 입력 수집 — 개발 계획(D-06) / 개발설계(D-08) md + 선행 요구사항·화면 식별자
② AEGIS 디스패치 (읽기·검색만)
   - 추적 키 grep 조인: REQ/AC ↔ SCR ↔ TSK(개발계획) ↔ LOGIC/TBL/QRY/API-NN(개발설계) ↔ 구현 FQN(후행)
     · 하나라도 비면 '누락', 키 불일치면 '모순'
   - D-06 §9 게이트: 순환 의존 0 · TE·σ 공식 일치(임의값 0) · 임계경로 slack=0 · 마일스톤 4단계 게이트 · 가용성/자원 배치 가정 명시
   - D-08 §9 게이트: MyBatis 쌍 · 멀티테넌트 키 · APIResult · resultCode 모듈 범위 · ${databaseNeos}/${databaseErp} · dbPatch
   - 연계(둘 다일 때): 개발 계획 produces == 개발설계 항목 ID 집합 (계획↔설계 완결)
③ 검증 보고 산출 — 누락·모순·형식 위반 목록 + 게이트 통과/실패 매트릭스
④ 제안→확정 판정
   - 게이트 전수 통과 + 미확정(_(확인 필요)_) 0 → '확정 가능'
   - 위반·미확정 있으면 '보완 필요' + 항목별 사유
   - ※ 확정 행위 자체는 실무 개발담당자 또는 리더(Cell리더) — AEGIS·본 스킬은 판정만(변경·확정 안 함, D-06 §7)
⑤ 후속 — '확정 가능'이면 사람이 status 제안→확정. 개발 계획 확정 → 개발설계(D-08) 착수 / 개발설계 확정 → 코딩·검수(D-11) 착수
```

## 4. 출력

| 산출 | 위치 |
|---|---|
| 정합성 검증 보고 .md | `프로젝트/PRJ-NNNN_*/05_산출물/{날짜}-{주제}-정합성검증.md` 또는 대상 모듈 작업 폴더 |
| 게이트 통과/실패 매트릭스 | 검증 보고 §매트릭스 |
| 제안→확정 판정 | 검증 보고 §판정 (확정 가능 / 보완 필요) |

## 5. 디스패치 예시

> 사용자: "법무 사건비용 개발 계획 검증해줘"

→ ① `Amaranth10/법무관리(LTE)/history/plans/lte-사건비용-개발계획.md` 수집 → ② AEGIS 디스패치(추적 키 조인 + D-06 §9 게이트) → ③ 검증 보고 → ④ '확정 가능' 판정 → ⑤ 실무자/리더가 status 제안→확정 → BLUEPRINT(개발설계 D-08) 착수.
