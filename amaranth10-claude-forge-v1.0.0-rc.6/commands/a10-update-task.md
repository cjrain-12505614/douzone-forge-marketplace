진행 중인 고도화 건의 상태(진행률·단계·이슈)를 업데이트하고 `_current.md`에 반영한다.

## 실행 순서

**1단계 — 대상 확인**
`$ARGUMENTS`가 있으면 해당 모듈의 `tasks/_current.md`와 관련 `tasks/enhancements/` 파일을 읽는다.
`$ARGUMENTS`가 없으면 현재 컨텍스트에서 모듈명과 고도화 건을 추론한다.

**2단계 — 업데이트할 항목 확인**
사용자에게 업데이트할 내용을 확인한다. 이미 대화 중 언급된 내용은 건너뛴다.
- 상태 변경 (검토중 / 기획중 / 개발중 / QA / 완료 / 보류)
- 진행률 (0~100%, 또는 기획X%·개발X%)
- 완료된 태스크 체크
- 신규 이슈 또는 해결된 이슈
- 완료 예정일 변경

**3단계 — 파일 업데이트**
변경 내용을 아래 파일에 반영한다.
- `tasks/_current.md` — 진행률, 상태, 날짜 업데이트
- `tasks/enhancements/{파일명}.md` — 태스크별 상태 업데이트

변경 시 파일 상단의 "마지막 업데이트" 날짜를 오늘로 갱신한다.

**4단계 — 완료 보고**
업데이트 결과를 요약한다.
예: "✅ '가온 요구사항 대응' 업데이트 완료 (79% → 83%). 타임시트 기준 개선 완료, 상담료 프로세스 80% 반영."

## 완료 처리 시 추가 작업

상태가 '완료'로 변경되면:
1. `tasks/_current.md`에서 진행 중 섹션 → 최근 완료 섹션으로 이동
2. 관련 context 파일 업데이트 필요 여부 확인 → 필요 시 `/update-context` 실행 안내
3. `history/_timeline.md` 완료 날짜 기록
